Import Observations

Bring an eBird export or any other spreadsheet of sightings into Tripper.

Getting an eBird Export

From ebird.org/downloadMyData, request "My eBird Data" — eBird emails you a link to a file usually named MyEBirdData.csv. That file works directly with Tripper's import — no editing required.

Required Columns

Every row needs a date and a way to identify the species (either a species code, or a common/scientific name matched against your downloaded species catalog).

Column Mapping

Headers are matched automatically, case- and punctuation-insensitive — Common Name, common_name, and COMMON-NAME all resolve the same way. If your file uses different column names (e.g. Bird instead of Species), Tripper's import screen shows a column mapping section after you pick a file — one dropdown per recognized field, pre-filled wherever auto-detection already found a match. Pick the right column for anything left unmapped, or leave it as Auto-detect/Not mapped if the field doesn't apply.

Supported File Types

CSV Comma-separated values — the only format currently supported. Export Excel/Sheets files as CSV first.

What Happens During Import

  • Rows are processed oldest-first by observation date.
  • Exact duplicates (same species, date, and location, for the same trip) are skipped automatically — safe to re-run an import if it's interrupted partway through.
  • Large files are imported in batches rather than as one all-or-nothing operation, so an interrupted import keeps whatever was already saved.
  • "Lifer" status (first-ever sighting of a species at each location scope) is recalculated once, after all rows are imported.
  • Imported observations sync to your account the same as any other observation.
💡 Tip: Rows with an unrecognized species or an unparseable date are skipped and listed as errors after the import finishes — everything else still imports normally.
